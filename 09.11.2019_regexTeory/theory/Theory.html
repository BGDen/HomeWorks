1

Опишите работу трех основных флагов в регулярных выражениях.

Ответ:

Флаги регулярных выражений задают высокоуровневые правила соответствия шаблонам. В отличие от остальной грамматики регулярных выражений, флаги указываются не между символами слэша, а после второго из них. В языке JavaScript поддерживается три основных флага и три типичных:

    Основные :
  • g - указывает, что поиск должен быть глобальным, т.е. должны быть найдены все соответствия с шаблоном поиска, без него – только первое.
  • i - указывает, что поиск по шаблону должен быть нечувствителен к регистру символов, нет разницы между A и a .
  • m - выполняет поиск по шаблону в многострочном режиме. Если строковое выражение, в котором выполняется поиск, содержит символы перевода строк, то в этом режиме якорные символы ^ и $, помимо того, что они соответствуют началу и концу всего строкового выражения, также соответствуют началу и концу каждой текстовой строки.
let re = /"([^"]*)"/g ;
    Типичные :
  • u - включает полную поддержку юникода, разрешает корректную обработку суррогатных пар (добавлено в ES6/ES2015) .
  • s - текст трактуется как одна строка, в этом случае метасимволу . (точка) соответствует любой одиночный символ, включая символ новой строки (новое в ES2018) .
  • y - выполняет поиск начиная с символа, который находится на позиции lastIndex и только в lastIndex (не ранее или позже в строке) .
Флаги можно комбинировать : let re = /^[ \s]+|[ \s]+$/gi ;

В чем отличие классов \w \W \s \S \d \D друг от друга?

Ответ:

Символьный класс – это специальное обозначение, которое соответствует любому символу из определённого набора.
У классов \d, \w и \s есть отрицания – \D, \W и \S соответственно.
Например , оператор \D будет искать соответствия противоположенные \d.

\s - совпадает с любым пробельным символом: пробел, табуляция, символ новой строки и пробелы Unicode
\S - совпадает с любым символом, который не является пробелом

\d - совпадает с любым числом, эквивалентно [0-9]
\D - совпадает с любым символом, который не является числом, эквивалентно [^0-9]

\w - совпадает с любым буквенно-числовым символом, эквивалентно [A-Za-z0-9]
\W - совпадает с любым символом, который не является буквенно-числовым значением, эквивалентно [^A-Za-z0-9]

Эти полезные классы используются для быстрого решения наиболее распространенных задач веб-разработки.

Что такое квантификаторы в регулярных выражениях?

Ответ:

Квантификаторы - это метасимволы позволяющие задать количество повторений символа или группы символов в строке.

Доступны нам следующие квантификаторы:

* — любое кол-во вхождений предшествующего символа /^\d*$/ ;
+ — одно или более вхождений предшествующего символа /^\d+$/ ;
? — указывает на необязательность предшествующего символа /^\d?$/ ;
{n} — точное кол-во вхождений предшествующего символа /^\d{3}$/ ;
{n, } — минимальное кол-во вхождений предшествующего символа /^\d{3,}$/ ;
{n, m} — возможно кол-во вхождений предшествующего символа от n до m /^\d{3,5}$/ ;
Используя их, мы можем очень гибко валидировать входящие данные.

Квантификаторы ( * + {}) – это «жадные» операторы, потому что они продолжают поиск соответствий, как можно глубже – через весь текст.
В «жадном» режиме (по умолчанию) квантификатор повторяется столько раз, сколько это возможно.
let regexp = /".+"/g;

«Ленивый» режим означает – повторять квантификатор наименьшее количество раз.
Обычно знак вопроса ? сам по себе является квантификатором, но, если он добавлен после другого квантификтора (или даже после самого себя), он получает
другое значение – меняет режим совпадения с жадного на ленивый.
let regexp = /".+?"/g;

Что такое группировка и обратные ссылки в регулярных выражениях?

Ответ:

Круглые скобки () имеют в регулярных выражениях несколько значений.
Одно из них – группировка отдельных элементов в одно подвыражение (захватываемоя группа), так что элементы при использовании специальных символов |, *, +, ? и других, рассматриваются как одно целое.
Например , шаблон
/(ab|cd)+|ef)/   соответствует либо строке «ef», либо одному или более повторений одной из строк «ab» или «cd».

Обратные ссылки предоставляют удобный способ идентификации повторяющегося символа или подстроки в строке.
Если входная строка содержит несколько экземпляров произвольной подстроки, можно найти первое вхождение с помощью группы записи, а затем использовать обратную ссылку для поиска последующих вхождений подстроки.
Обратная ссылка – \n где n – это порядковое положение захватываемой группы в регулярном выражении.
Например , в регулярном выражении   /(one(two) (three))(four)\1/
\1 ссылается на «one»,
\2 на «two»,
\3 на «three»,
\4 на «four».

Опишите методы строк, которые используют регулярные выражения.

Ответ:

Строки поддерживают четыре метода, использующие регулярные выражения.
Простейший из них - метод search(). Он принимает в качестве аргумента регулярное выражение и возвращает либо позицию первого символа найденной подстроки, либо -1, если соответствие не найдено.
Например , следующий вызов вернет 4 :
let re = "JavaScript".search(/script/i); // 4

Mетод replace() выполняет операцию поиска с заменой. Он принимает в качестве первого аргумента регулярное выражение, а в качестве второго - строку замены. Метод отыскивает в строке, для которой он вызван, соответствие указанному шаблону.
Следующий пример использует метод replace(), чтобы поменять местами слова (символы) в строке. Для замены текста скрипт использует обозначения $1 и $2 для обозначения первого и второго совпадения скобочного выражения.
Например :
let re = "John Smith".replace(/(\w+)\s(\w+)/, "$1, $2");   // Smith John

Mетод match() - он принимает в качестве единственного аргумента регулярное выражение и возвращает массив, содержащий результаты поиска.
Если в регулярном выражении установлен флаг g, метод возвращает массив всех соответствий, присутствующих в строке.
Например :
let re = "1 плюс 2 равно 3".match(/\d+/g); // ["1", "2", "3"]

Mетод split() - разбивает строку, для которой он вызван, на массив подстрок, используя аргумент в качестве разделителя.
Например :
let result = "12, 34, 56".split(/,\s*/); // [12, 34, 56]

Опишите методы самих регулярных выражений на совпадение и поиск данных в строке.

Ответ:

Mетод regexp.test(str) ищет совпадение и возвращает логическое (boolean)
значение true / false, в зависимости от того, находит ли он его.
Например :
let result = /love/i.test("I love JavaScript"); // true let result = /love/i.test("Bla-bla-bla"); // false

Mетод regexp.exec(str) проверяет, соответствует ли строка регулярному выражению, возвращает ту часть строки, которая соответствует выражению. Если соответствий нет, то возвращается значение null.
Например :
let result = /hello/.exec("hello world!"); // hello let result = /hello/.exec("beautifull wheather"); // null

Также результатом является массив подстрок, соответствующих заданному образцу:
let result = /(\d+).(\d+).(\d+)/.exec("Я родился 27.03.1217");
 console.log(result[0]); // 27.03.1217
console.log(result[1]); // 27
console.log(result[2]); // 03
console.log(result[3]); // 1217
